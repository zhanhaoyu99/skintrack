# 皮肤管理App (SkinTrack) 项目策略

## 产品定位
**一句话**: 每天拍一张脸，AI追踪皮肤变化，告诉你护肤品到底有没有用。

**核心价值主张**: 不再只能"感觉"护肤品有没有效，而是用数据看得见。

**核心差异化**: 所有竞品都只做"单次检测"，SkinTrack做"长期追踪 + 护肤品归因分析"。

**目标用户**: 18-35岁女性为主，有护肤习惯，愿意为护肤品花钱但不确定效果的人群。

**产品形态**: KMP App (Android首发 + iOS跟进)

## MVP功能分级
### P0 核心 (V1.0)
1. **每日拍照分析** — AI自动识别痘痘、黑头、毛孔、红血丝、肤色均匀度，综合评分
2. **时间线趋势图** — 周/月维度皮肤指标变化曲线 + 前后对比图
3. **护肤品记录** — 扫码快速录入 + 手动搜索 + 历史复用
4. **拍照标准化引导** — 固定角度框 + 光线检测 + 无妆提醒

### P1 关键差异化 (V1.0)
5. **归因分析报告** — AI关联护肤品使用与皮肤指标变化，因果推断
6. **打卡提醒 + 激励** — 连续打卡奖励，"再坚持3天解锁报告"

### P2 增强 (V1.5-V2.0)
7. 饮食/睡眠/姨妈期记录
8. 护肤品效果排行榜（众包数据）
9. 社区功能（分享对比图、产品评价）

### P3 远期 (V3.0)
10. 皮肤科连接（报告导出/分享给医生）

## 商业模式
| 收入源 | 时间线 | 占比 |
|--------|--------|------|
| 会员订阅 19.9元/月 | 上线即可 | 60% |
| 护肤品CPS导购分佣 | V1.5+ | 15% |
| 品牌排行榜投放 | V2.0+ | 20% |
| 皮肤科/医美导流 | V3.0+ | 5% |

**付费转化关键**: 前14天免费→第14天弹出归因报告→免费看摘要→详细报告需付费→沉没成本驱动转化

**盈亏平衡**: ~150个付费会员

**单用户月成本**: ~1-3元 (AI API + 存储)，毛利率>85%

## 核心用户旅程
1. 注册 → 微信授权 + 简单问卷（性别/年龄/肤质）
2. 每日拍照 → 对准人脸框 → AI分析 → 当日评分
3. 记录护肤品 → 一键复用昨天 / 修改
4. 查看趋势 → 周/月曲线图 + 前后对比图
5. 归因报告 → "什么让你变好/变差"
6. 分享 → 生成精美前后对比卡片

## 拍照标准化方案（核心难点）
- 人脸对齐框 — 固定角度
- 光线质量检测 — AI判断合格否
- 无妆提醒 — 引导素颜
- 参考时间引导 — 建议每天同一时间（如早上洗脸后）

## 里程碑规划（基于KMP技术方案）
| 阶段 | 时间 | 内容 |
|------|------|------|
| M0: 项目搭建 | 2周 | KMP初始化 + Room/Ktor/Supabase配置 + 基础架构 |
| M1: 核心功能 | 4周 | 相机+人脸引导(expect/actual) + AI分析 + 护肤品记录 + 时间线图表 + 归因报告 |
| M2: 变现功能 | 2周 | 会员订阅(微信支付+StoreKit 2) + 免费/付费门控 + 分享卡片 |
| M3: 测试上线 | 2-4周 | 内测20-30人 + Android先发 + iOS提交审核 |

## 增长策略
- **核心渠道**: 小红书前后对比图种草（天然传播内容）
- **裂变**: 分享对比图自带App链接、邀请奖励、闺蜜护肤PK
- **内容营销**: 护肤品实测数据评测、换季护肤指南

## 开发节奏
- MVP先行，功能最小可用即交付
- Android先验证，iOS跟进适配
- 每个里程碑端到端完成
- 预计代码复用率 85-90%（仅相机/通知/支付/图片压缩需平台代码）
