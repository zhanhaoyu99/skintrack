# 拍照页 CameraScreen

> 设计稿: `.figma-mockups/05-camera.html` (3 态: 取景/结果/权限请求)

## 入口
- 底部导航 FAB "拍照" 按钮
- Dashboard 拍照提醒卡 "去拍照" 按钮
- Dashboard 空状态 "拍第一张自拍" 按钮
- 全屏页面（非 Tab 内嵌），带返回导航

## 状态 1 — 取景模式

```
┌─────────────────────────┐
│ ← (暗色)        ⚡ ⟳    │ ← TopBar: 返回/闪光灯/翻转
│                         │
│                         │
│    ┌ ─ ─ ─ ─ ─ ─ ┐     │
│    │               │     │ ← 人脸引导椭圆（虚线描边）
│    │   (预览画面)    │     │
│    │               │     │
│    └ ─ ─ ─ ─ ─ ─ ┘     │
│                         │
│   将面部置于框内          │ ← 居中提示文字（白色）
│                         │
│  🔄    ◉    ⚡          │ ← 底部: 翻转/拍照按钮/闪光灯
└─────────────────────────┘
```

### 规格
- **背景**: 全黑（camera.background）
- **人脸引导框**: 椭圆形虚线描边 (1.5px dashed), 白色 20% alpha (`rgba(255,255,255,0.2)`)
- **提示文字**: "将面部置于框内 - 保持正面"，白色居中
- **拍照按钮**: 76dp 圆形，白色实心，3.5dp 描边
- **辅助按钮**: 翻转摄像头、闪光灯切换，暗色半透明圆形背景
- **TopBar**: 返回箭头（白色），闪光灯/翻转图标

## 状态 2 — 分析结果

```
┌─────────────────────────┐
│                         │
│    [拍摄的照片预览]       │ ← 上半部分: 肤色渐变背景 + 人脸轮廓
│      ◎ 82分             │ ← 底部居中 ScoreRing 悬浮
│                         │
├─────────────────────────┤
│  皮肤状态很棒哦!         │ ← 状态描述 (19sp Bold)
│  痘痘明显减少了...        │ ← AI 简评 (13sp)
│                         │
│  [85][78][82][75][80]   │ ← 迷你指标行 (5 维)
│  痘痘 毛孔 均匀 泛红 水润 │
│                         │
│  🔥 连续打卡 7 天        │ ← 打卡连续 pill (Rose 色)
│                         │
│ [继续拍照]  [查看详情]    │ ← 双按钮: outlined + primary
└─────────────────────────┘
```

### 规格
- **照片预览**: 上半区域，肤色渐变模拟背景，半透明人脸轮廓
- **ScoreRing**: 悬浮在照片底部中央，小尺寸，显示分数 + "总分"
- **状态标题**: 根据评分档位显示不同文案
- **AI 简评**: 一句话摘要（13sp）
- **迷你指标**: 5 个方块紧凑排列，每个显示分值 + 标签
- **打卡 pill**: 🔥 图标 + "连续打卡 N 天"，Apricot 色渐变圆角条 `linear-gradient(135deg, apricot-50, #FFF8F0)`
- **双按钮**: "继续拍照"(outlined) + "查看详情"(primary)

### 规格
- **图标**: 粉色圆形背景 (rose-100) + 相机图标 (rose-400)
- **标题**: "来拍张自拍吧"（22sp Bold）
- **描述**: "我们需要相机权限来帮你记录肌肤状态"
- **权益列表**: 3 条，绿色勾 (primary) + 文字 (14sp)
  - "AI 智能分析五大肌肤指标"
  - "追踪每天的肌肤变化趋势"
  - "照片仅存储在本地设备中"（隐私保证）
- **主按钮**: "允许使用相机"（primary 全宽）
- **次要**: "下次再说"（文字链接，返回上一页）

## 拍照流程

```
取景 → 点击拍照 → 压缩图片 → AI 分析(异步) → 保存 Room → 显示结果
                                  ↓ 失败
                            保存原始记录（pending 状态）
                            后台队列重试
```
