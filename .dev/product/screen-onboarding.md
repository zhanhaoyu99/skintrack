# 引导页 OnboardingScreen

> 设计稿: `.figma-mockups/01-onboarding.html` (4 屏)

## 触发条件
- 首次安装 App（DataStore 标记 `hasSeenOnboarding = false`）
- 完成后设置 `hasSeenOnboarding = true`，不再显示

## 页面结构

### 公共布局
```
┌─────────────────────────┐
│      Status Bar          │
├─────────────────────────┤
│                         │
│    [圆形插画区域]         │ ← 240x240dp，多层叠圆 + SVG 图标 + 浮动装饰点
│    标题（2行）            │ ← 28sp Bold
│    描述文案               │ ← 15sp，onSurfaceVariant
│                         │
├─────────────────────────┤
│  ● ○ ○ ○               │ ← 分页指示器（活跃态 28dp 宽圆角条）
│  [主按钮]                │ ← 全宽 primary 按钮
│  跳过                    │ ← 14sp，仅 P1-P3 显示
└─────────────────────────┘
```

### Page 1 — 记录你的肌肤变化
- **插画**: 薄荷绿圆底 + 人脸+相机 SVG 图标 + 玫瑰粉装饰圆
- **标题**: "记录你的肌肤变化"
- **描述**: "每天一张自拍，AI 帮你追踪痘痘、毛孔、肤色等指标，见证皮肤一天天变好"
- **按钮**: "下一步"
- **底部**: "跳过"

### Page 2 — 管理你的护肤方案
- **插画**: 玫瑰粉圆底 + 护肤瓶罐 SVG 图标 + 薄荷绿装饰圆
- **标题**: "管理你的护肤方案"
- **描述**: "记录每天用了哪些护肤品，让数据告诉你什么组合最适合你的肌肤"
- **按钮**: "下一步"
- **底部**: "跳过"

### Page 3 — AI 帮你找到最佳护肤方案
- **插画**: 薰衣草紫圆底 + 柱状图+趋势线 SVG 图标 + 玫瑰粉装饰圆
- **标题**: "AI 帮你找到最佳护肤方案"
- **描述**: "哪个精华让你皮肤变好了？AI 为你分析每个产品的真实效果，科学护肤不踩坑"
- **按钮**: "下一步"
- **底部**: "跳过"

> 注意: HTML 设计稿 P3 的指示器有 5 个点，这是设计稿制作过程中的临时偏差。
> 最终实现应使用 4 个点（对应 4 页），各页指示器为 `dots[currentPage].active`。

### Page 4 — 肤质选择
- **标题**: "你的肤质是？"（26sp Bold）
- **副标题**: "选择肤质帮助 AI 更准确地分析"（14sp）
- **选项列表**（单选，垂直排列，10dp 间距）:

| 肤质 | 图标底色 | 描述 |
|------|---------|------|
| 油性肌肤 | Apricot 渐变 | T区容易出油，毛孔偏大 |
| 干性肌肤 | 蓝色渐变 | 容易干燥紧绷，需要保湿 |
| 混合肌肤 | Mint 渐变 | T区偏油，两颊偏干 |
| 敏感肌肤 | Rose 渐变 | 容易泛红、刺痛，屏障脆弱 |
| 中性肌肤 | Lavender 渐变 | 水油平衡，状态稳定 |

- **选中态**: 2px primary 边框 + `linear-gradient(135deg, mint-50, #F0FAF6)` 背景 + primary 实心勾选圆 (22dp)
- **未选中态**: 1.5px outlineVariant 边框 + surface 背景 + 22dp 空心圆 (outlineVariant 描边)
- **选项卡片**: 16dp padding, lg 圆角, 10dp 间距, shadow-xs
- **图标容器**: 44dp 圆角方形 (13dp radius), 各肤质对应渐变底色
- **按钮**: "开始使用" (全宽 primary)
- **无"跳过"按钮**（必须选择肤质）

## 交互逻辑
1. 左右滑动切换页面，或点击"下一步"
2. P1-P3 可跳过，直接进入 AuthScreen
3. P4 肤质选择完成后进入 AuthScreen
4. 肤质信息保存到 UserPreferences（DataStore 或 Room）
5. 分页指示器：活跃态为 28dp 宽圆角条 `primary`，非活跃为 8dp 圆点 `outlineVariant`
