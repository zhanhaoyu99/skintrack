# 工作流：进度报告

## 触发命令
`/app status` — 全面展示项目当前状态

## 执行流程

### 第1步：加载全部记忆
必须读取以下文件：
```
app-progress.md    — 进度主文件
app-issues.md      — 问题追踪
app-decisions.md   — 架构决策
app-strategy.md    — 项目策略
app-learnings.md   — 经验积累
```

### 第2步：扫描代码仓库
1. 检查项目目录结构（如已初始化）
2. 统计：
   - 代码文件数量（按模块）
   - 测试文件数量
   - 最近修改的文件
3. 检查构建状态（如可执行）

### 第3步：生成进度报告

**输出格式：**


### 第4步：智能建议
根据当前状态生成下一步建议：
- 如果有阻塞问题 → 优先解决阻塞
- 如果当前里程碑有未完成任务 → 继续当前里程碑
- 如果当前里程碑完成 → 建议开始下一里程碑
- 如果有技术债务积累 → 建议安排重构
- 如果长时间未写测试 → 建议补充测试

---

## 注意事项
- 报告要准确反映 app-progress.md 中的数据
- 不要臆造进度，如果记忆文件中没有记录就标注"未记录"
- 如果代码仓库存在但记忆文件落后，提示用户需要同步
- 始终包含下一步建议
